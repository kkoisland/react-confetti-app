# React Confetti サンプルアプリ 設計書

## プロジェクト概要

このプロジェクトは、React Confetti ライブラリの解説本のためのサンプルアプリケーションです。
読者が本を読んで理解した後、実際のサンプルで「ああ、こういうことか」と確認できるように設計されています。

**執筆・実装の進め方**: 実装する → 節を書く → 次の節

## 技術仕様（簡潔版）

**技術スタック**:
- React 19.x + TypeScript 5.9.x + Vite
- Tailwind CSS（スタイリング）
- React Router v6（ルーティング）
- react-confetti（紙吹雪ライブラリ）
- Biome（リンター/フォーマッター）

**ルート構造**:
```
/                    → /basic へ自動リダイレクト
/basic               → 第2節 基本の紙吹雪
/countdown           → 第3節 タイマーによる自動制御
/toast               → 第4節 実務での演出パターン
/seasonal            → 第5節 エフェクトのカスタマイズ
/playground          → 第6節 パラメータの探求
```

**デザイン方針**:
- モダンで美しいUI（グラデーション背景、ガラスモーフィズム）
- 上部ナビゲーション + メインエリア（紙吹雪を最大限広く見せる）
- ダーク/ライトモード対応（切り替えボタン付き）
- レスポンシブデザイン
- 各ページごとに異なるグラデーション背景で個性を演出

**カラーテーマ**:
- ベース文字色: `text-gray-800` (全体に適用、真っ黒より読みやすい)
- アクセントカラー: `text-indigo-900` (ホバー時、濃い紺色)
- 背景色: `bg-white` (ライトモード)
- ダークモード: 後で `dark:bg-gray-900` を追加予定

**レスポンシブ対応の方針**:
1. 基本レイアウト（上部ナビなど）：最初からレスポンシブ対応
2. 各ページ（/basic, /countdown等）：デスクトップ優先で実装、見た目が揃ってから調整

## 第6節 パラメータの探求 (`/playground`)

### 節の構成案
- 6.1 パラメータの解説
- 6.2 パフォーマンスとのバランス
- 6.3 インタラクティブな調整UI

### 機能
- インタラクティブなプレイグラウンド
- リアルタイムで調整結果が反映される

**調整可能なパラメータ**：
- **スライダー**：
  - numberOfPieces（粒子数：50-500、刻み幅=10）
  - gravity（重力：0-0.3、刻み幅=0.01）
  - wind（風：-0.1-0.1、刻み幅=0.01）
  - initialVelocityX（横初速：-10-10、刻み幅=1）
  - initialVelocityY（縦初速：-20-0、刻み幅=1）
- **カラーピッカー**：3色まで選択可能
- **recycle**：常に `true`（永遠に降り続ける、固定）

**その他の機能**：
- **プリセットボタン**：第5節の5つのテーマ（桜、雪、紅葉、星、クリスマス）をワンクリック適用
- **リセットボタン**：デフォルト値に戻す（numberOfPieces=200, gravity=0.1など）
- **パラメータ値の表示**：各スライダーの隣に現在値を表示
- **コードスニペット表示**：画面下部に水平中央配置、リアルタイム更新、コピーボタン付き
- **コピー成功通知**：コピーボタンを押すとテキストが「コピー済み✓」に変わる（2秒後に戻る）
- **初期状態**：ページを開いたときからデフォルト設定で紙吹雪が降っている

**コードスニペット表示例**：
```jsx
<Confetti
  numberOfPieces={200}
  gravity={0.1}
  wind={0}
  colors={['#FF0000', '#00FF00', '#0000FF']}
/>
```
※ recycle=trueはデフォルト値なので省略可

### デザイン
- **背景**: `bg-gradient-to-br from-white via-green-100 to-cyan-200`
- **ガラスモーフィズム**: `bg-white/30 backdrop-blur-md`
- **レイアウト**：
  - **上部**：調整UI（3行構成）
    - 1行目：スライダー3つ（numberOfPieces, gravity, wind）横並び
    - 2行目：スライダー2つ（initialVelocityX, initialVelocityY）横並び
    - 3行目：カラーピッカー3つ＋プリセットボタン5つ＋リセットボタン、横並び
  - **下部**：コードスニペット表示（水平中央配置、bg-gray-900、z-index高、紙吹雪の上に重なる）
  - **全体**：紙吹雪が画面全体に降る

### 他のUIパターンへの応用

**コンポーネント種類ごとの活用例**：
- **モーダル（ダイアログ）**：会員登録完了モーダル、レベルアップモーダルなど
- **フォーム送信成功時**：送信ボタンを押して成功レスポンスが返ってきたタイミング
- **ローディング完了時**：時間のかかる処理（データインポート、ファイルアップロードなど）が完了したタイミング

**シチュエーション例**：
ゲームの正解時やレベルアップ時、購入完了時、会員登録完了時、目標達成時、タスク完了時など、ユーザーにとって嬉しい瞬間に紙吹雪を使うことで、体験をより印象的にできます。

**書く内容**：
- 各パターンでの実装のヒント（どのタイミングでstateを切り替えるか）
- 適切な使用場面と避けるべき場面
- ユーザー体験を損なわないための配慮（頻度、タイミング）

### Canvas Confettiについて
React Confettiはシンプルで使いやすいですが、より複雑な演出が必要な場合は
**Canvas Confetti**（https://github.com/catdad/canvas-confetti）も検討してください。

**Canvas Confettiでできること**：
- 花火のような爆発演出
- 複数の発射位置を指定
- より細かい制御（角度、速度、形状など）
- 時間差での複数発射

ただし、設定が複雑になるため、まずはReact Confettiで十分です。
必要に応じて、段階的にCanvas Confettiへ移行することもできます。

### セットアップ手順（実装時の参考用）

#### 0. スタイリング方針
- **index.css**: すべての必要なCSSをここに集約
- **インラインスタイル**: 必要に応じて使用
- **方針**: CSSファイルを分散させず、index.cssに統一

#### 1. Tailwind CSSのインストールと設定

**インストール**：
```bash
pnpm add -D tailwindcss @tailwindcss/vite
```

**src/index.cssを編集**：
```css
@import "tailwindcss";
```
既存のCSSは削除して、この1行のみにする。

**vite.config.tsを編集**：
```ts
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [react(), tailwindcss()],
})
```

**クリーンアップ**：
- `src/App.css` を削除（不要）
- `src/App.tsx` をシンプルなHello Worldに変更

#### 2. React Routerのインストールと設定
```bash
pnpm add react-router-dom
```

#### 3. react-confettiのインストール
```bash
pnpm add react-confetti
pnpm add -D @types/react-confetti
```

#### 4. 基本レイアウト（上部ナビ + ルーティング）

**重要**: この段階でレスポンシブ対応を含めて実装します。各ページ（/basic等）はデスクトップ優先で実装するため、レイアウトだけはここでしっかり作ります。

**実装のポイント**：
- 上部ナビゲーションバー（/basic, /countdown, /toast, /seasonal, /playground）
- メインエリア（各ページのコンテンツ、画面全体を使える）
- レスポンシブデザイン（モバイルではハンバーガーメニュー、デスクトップでは横並びメニュー）
- 紙吹雪が画面全体に広がるよう、メインエリアは最大限の広さを確保

#### 5. ダークモード切り替え

サンプルアプリをより見やすくするため、ダーク/ライトモード切り替え機能を実装します。
これはReact Confettiの機能ではなく、アプリ全体のUI改善のためのおまけ機能です。

**実装のポイント**：
- Tailwindの `dark:` クラスを使用
- 上部ナビの右上に切り替えボタンを配置
- `localStorage` で設定を保存（オプション）
- 各ページのグラデーション背景にダークモード版を用意

本書では詳細な説明は省略しますが、実装されたサンプルコードを参考にしてください。
